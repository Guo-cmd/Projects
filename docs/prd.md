# 需求文档

## 1. 应用概述

### 1.1 应用名称
番茄钟

### 1.2 应用描述
一个前端实现的番茄钟时间管理工具，帮助用户通过番茄工作法提高专注力和工作效率。用户可以设置工作时长、休息时长和长假时长，应用会进行倒计时并记录完成的番茄钟数量。界面简洁美观，所有数据存储在浏览器本地。支持移动端访问，可作为PWA应用添加到手机主屏幕并离线使用。

## 2. 用户与使用场景

### 2.1 目标用户
需要时间管理和专注力提升的用户，包括学生、职场人士、自由职业者等。

### 2.2 核心使用场景
- 用户需要专注工作或学习一段时间
- 用户希望通过番茄工作法管理时间
- 用户需要记录自己的工作成果
- 用户在移动端使用番茄钟工具
- 用户希望将应用添加到手机主屏幕快速访问
- 用户在离线状态下使用应用

## 3. 页面结构与功能说明

### 3.1 页面结构
```
番茄钟应用
├── 主页面
│   ├── 倒计时显示区域
│   ├── 控制按钮区域
│   ├── 设置区域
│   └── 统计区域
```

### 3.2 主页面功能说明

#### 3.2.1 倒计时显示区域
- 显示当前倒计时剩余时间（格式：MM:SS）
- 显示当前状态（工作中/休息中/长假中）

#### 3.2.2 控制按钮区域
- 开始按钮：启动倒计时
- 暂停按钮：暂停或恢复倒计时（切换功能）
- 恢复按钮：重置当前倒计时到初始时长并回到初始状态
- 按钮状态规则：
  - 初始状态：开始按钮可用，暂停按钮和恢复按钮禁用（呈现灰色）
  - 点击开始后：开始按钮禁用，暂停按钮和恢复按钮变为可用状态
  - 点击暂停后：倒计时暂停，再次点击暂停则倒计时恢复
  - 点击恢复后：倒计时重置为当前状态的初始时长，开始按钮变为可用，暂停按钮和恢复按钮变为禁用

#### 3.2.3 设置区域
- 番茄钟时长设置：用户输入工作时长（单位：分钟）
- 休息时长设置：用户输入短休息时长（单位：分钟）
- 长假时长设置：用户输入长休息时长（单位：分钟）
- 保存设置按钮：保存用户设置的时长到浏览器本地存储

#### 3.2.4 统计区域
- 显示今日完成的番茄钟数量
- 显示总完成的番茄钟数量

#### 3.2.5 提醒音效
- 工作倒计时结束时播放提醒音效
- 休息倒计时结束时播放提醒音效

#### 3.2.6 移动端适配
- 应用在375px-430px等小屏设备上完美运行
- 界面元素自动调整布局和尺寸适应屏幕宽度
- 按钮和输入框大小适合触摸操作
- 文字大小在小屏设备上清晰可读

#### 3.2.7 PWA功能
- 用户可通过浏览器将应用添加到手机主屏幕
- 应用具备独立图标和启动画面
- 应用支持离线访问，在无网络状态下可正常使用核心功能
- 应用具备独立窗口运行能力

## 4. 业务规则与逻辑

### 4.1 倒计时规则
- 用户点击开始按钮后，根据当前状态开始倒计时
- 首次启动默认为工作状态，使用番茄钟时长
- 倒计时结束后自动切换状态：工作结束→休息开始，休息结束→工作开始
- 每完成4个番茄钟后，下一次休息使用长假时长，否则使用休息时长

### 4.2 数据存储规则
- 用户设置的时长保存在浏览器本地存储中
- 完成的番茄钟数量保存在浏览器本地存储中
- 今日完成数量在每日0点自动重置

### 4.3 默认时长
- 番茄钟时长：25分钟
- 休息时长：5分钟
- 长假时长：15分钟

### 4.4 UI设计原则
- 界面精致、现代，符合专注力工具特性
- 按钮状态变化清晰可见（可用/禁用状态区分明显）
- 整体视觉风格简洁、专注
- 移动端界面布局合理，操作便捷

### 4.5 PWA配置规则
- 应用包含manifest.json配置文件，定义应用名称、图标、主题色、显示模式等信息
- 应用注册Service Worker实现离线缓存策略
- 应用提供多种尺寸的图标资源
- 应用主题色与界面风格保持一致

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户输入非数字时长 | 提示用户输入有效数字 |
| 用户输入时长小于1分钟 | 提示用户时长不能小于1分钟 |
| 用户输入时长大于120分钟 | 提示用户时长不能超过120分钟 |
| 倒计时进行中关闭浏览器 | 下次打开时倒计时重置 |
| 浏览器不支持本地存储 | 使用默认时长，统计数据不保存 |
| 浏览器不支持Service Worker | 应用正常运行但无法离线使用 |
| 用户在离线状态下访问应用 | 应用从缓存加载，核心功能正常使用 |
| 小屏设备屏幕宽度低于375px | 界面元素继续缩放适配 |

## 6. 验收标准

1. 用户打开应用，看到倒计时显示为25:00，状态显示为「工作中」，开始按钮可用，暂停按钮和恢复按钮禁用呈现灰色
2. 用户点击开始按钮，倒计时开始递减，开始按钮禁用，暂停按钮和恢复按钮变为可用状态
3. 用户点击暂停按钮，倒计时暂停，再次点击暂停按钮，倒计时恢复
4. 倒计时结束后，播放提醒音效，状态自动切换为「休息中」，倒计时显示为05:00
5. 用户点击恢复按钮，倒计时重置为当前状态的初始时长，开始按钮变为可用，暂停按钮和恢复按钮变为禁用
6. 用户完成一个完整的工作倒计时，统计区域显示今日完成1个番茄钟
7. 用户在安卓手机（屏幕宽度375px-430px）上打开应用，界面元素布局合理，按钮和文字大小适合触摸操作
8. 用户通过浏览器菜单将应用添加到手机主屏幕，主屏幕显示应用图标
9. 用户点击主屏幕图标启动应用，应用以独立窗口运行
10. 用户在离线状态下打开应用，应用正常加载并可使用倒计时功能
